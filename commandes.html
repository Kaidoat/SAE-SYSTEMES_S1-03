Les différentes commandes utilisées


Création d'un compte administrateur


Pour créer un compte administrateur, on utilise la commande sudo adduser adminGr06_Binome03, où "Gr06" représente le groupe 106 et "Binome03" le groupe numéro 3. Cependant, Linux impose des restrictions sur les caractères autorisés dans les noms d’utilisateurs.

Afin de contourner cette limitation et permettre la création du compte malgré ces caractères, il est nécessaire d’activer l’option --allow-bad-names en exécutant la commande sudo adduser --allow-bad-names suivi de adminGr06_Binome03. Cela force le système à accepter des noms d’utilisateurs qui ne respectent pas les conventions habituelles de Linux. Après avoir entré cette commande, le système demande de saisir un mot de passe pour l’utilisateur. Il faut le taper deux fois pour confirmation.

Ensuite, la configuration de l’administrateur se poursuit en renseignant les informations suivantes : nom, numéro de chambre, téléphone professionnel, téléphone personnel, ainsi que d’autres détails optionnels. Création de deux utilisateurs BinomeYY_1 et BinomeYY_2


Nous créeons les deux utilisateurs en utilisant la commandesudo adduser --allow bad-names, qui permet d'accepter des noms d'utilisateurs contenant des caractères normalement interdits par Linux.

Pour le premier utilisateur, nous exécutons :

sudo adduser --allow-bad-names Binome03_1

Pour le deuxième utilisateur :

sudo adduser --allow-bad-names Binome03_2


Cette commande crée chaque utilisateur en demandant un mot de passe, puis des informations supplémentaires comme le nom complet, le numéro de téléphone, et la chambre.

Une fois la création terminée, les deux utilisateurs peuvent être utilisés normalement sur le système.


Pour vérifier que les utilisateurs ont bien été créés, on peut utiliser la commande cat/etc/passwd | grep Binome03

La première partie de la commande, cat/etc/passwd, permet d'afficher le contenu du fichier /etc/passwd qui contient la liste de tous les utilisateurs enregistrés sur le système.

Ensuite, le symbole "|" est utilisé pour envoyer cette sortie à une autre commande. Ici, on utilise grep Binome03, qui permet de filtrer les résultats et d'afficher uniquement les lignes contenant "Binome03"

Cela permet de s'assurer rapidement que les utilisateurs ont bien été créés en verifiant leur présence dans le fichier.

Création d'un groupe

Afin de créer un groupe nous utilisons la commande sudo groupadd Binome03


Pour vérifier que le groupe a bien été créé après l'exécution de la commande sudo groupadd Binome03, on peut soit consulter le fichier /etc/group, soit afficher la liste complète des groupes du système.

La première méthode consiste à rechercher le groupe directement dans le fichier en utilisant la commande cat/etc/group | grp Binome03.

Si le groupe existe, il apparaîtra dans la sortie. La seconde méthode permet d'afficher la liste complète des groupes enregistrés sur le système avec la commande getent groupe, et si l'on souhaite filtrer un groupe spécifique, on peut utiliser getent group Binome03. Si le groupe est bien présent, la ligne correspondante s'affichera, confirmant ainsi sa création.

Pour ajouter un utilisateur à un groupe sous Linux, on utilise la commande usermod -aG, où -a signifie "ajouter" sans supprimer les autres groupes de l'utilisateur, et -G spécifie le groupe cible.

Par exemple, pour ajouter l’utilisateur Binome03_1 au groupe Binome03, on exécute sudo usermod -aG Binome03 Binome03_1. On répète la commande pour tout autre utilisateur à ajouter.

Une fois l’opération effectuée, il est important de vérifier que l’utilisateur a bien été ajouté au groupe. Pour cela, on peut utiliser groups Binome03_1, qui affichera la liste des groupes auxquels l’utilisateur appartient.

Une autre méthode consiste à rechercher directement le groupe dans le fichier /etc/group avec grep Binome03 /etc/group. Cette commande affichera la ligne du groupe et la liste des utilisateurs qui y sont associés, confirmant ainsi l’ajout correct de l’utilisateur.
